Welcome to the Decarb Green Systems Architecture Model.

To get started, navigate through the model using the index on the left and the breadcrumb trail above. Zoom in and out of the diagrams using the zoom controls on the diagram or the shortcut ctrl+(+/-). The Reset button resets the zoom back to 100%. To toggle the diagram key, click Show Diagram Key.

Search the model by typing the name of the element (e.g., project, interface, etc.) you are looking for and choose the correct match from the list to open its specification page. This page displays all information relating to the element, including links to the diagram(s) where it appears. Clicking on one of these links will open that diagram with the element highlighted. The Clear button removes the highlighting.

The system architecture is modelled in two ways:

- The first is the System Breakdown Structure (SBS), which is the hierarchical decomposition of the system. It breaks the overall system into smaller, more manageable elements. The top level is the Fortescue Decarbonisation Portfolio. This is broken down using the following hierarchy: Program, System, Project, and Subsystem.

- The second is the System Definition (SD), which presents all the elements that exist in the SBS, and shows how they interface with each other, stakeholders, and external assets. The diagram is generic, meaning that physical assets are not modelled for specific locations, instead illustrating how they interface generally with each other and with the Decarb Green Systems.

The SD is also divided into individual diagrams for a selection of Decarb Green Systems. These can also be accessed by ctrl+clicking on the related block in the diagrams, denoted by a link symbol in the bottom-right of the block.
